1. Overview

Why a Browser Extension?

Phase 1 captures time from Google Workspace (Calendar, Gmail, Drive) and direct user input. However, Thrive's team works across dozens of web-based tools — Jira, Figma, Slack, Asana, Canva, social media platforms, and more. A browser extension provides universal coverage by tracking active time in any web application, without requiring individual API integrations for each tool.

One extension replaces dozens of integrations. Instead of building separate connectors for Jira, Figma, Slack, Asana, and every other tool, the extension tracks them all through a single mechanism: active tab monitoring with intelligent project mapping.

How It Works

  1. Passive tracking: The extension monitors which browser tab is active and for how long
  2. Smart clustering: Short tab visits are grouped into meaningful work sessions (e.g., 45 minutes across 12 Jira tabs = one "Jira project work" session)
  3. AI project mapping: The assistant maps URLs and page titles to Harvest projects using configurable rules + AI inference
  4. Daily summary: At end of day, the assistant presents tracked sessions as draft time entries for review
  5. User approval: Entries follow the same draft-and-approve workflow from Phase 1 — nothing reaches Harvest without user sign-off

2. Scope of Work

2.1  Chrome Browser Extension

Active Tab Monitoring

  • Track active tab URL, page title, and duration
  • Idle detection — automatically pause tracking when user is away (configurable: 2, 5, or 10 minutes)
  • Domain whitelist/blacklist — exclude personal sites (social media, banking, etc.)
  • Privacy-first design — data stays between user and platform, no keystroke logging, no screenshots

Extension Popup & Controls

  • Quick glance: today's tracked time by project
  • One-click manual time entry (for offline work or meetings)
  • Pause / resume tracking toggle
  • Current session indicator — shows active project mapping in real time
  • "This is not work" button — quickly exclude current activity

Project Mapping Engine

  • Rule-based mapping: URL patterns → Harvest projects (e.g., jira.com/*mediaworks* → "MediaWorks")
  • AI-assisted mapping: for unmapped URLs, the assistant infers the most likely project from page title and context
  • Learning: user corrections improve future mapping accuracy
  • Admin-configurable default rules for the organisation

2.2  Backend: Activity Ingestion

Data Pipeline

  • Secure API endpoint for extension data (encrypted, authenticated per user)
  • Activity aggregation — raw tab events compressed into sessions
  • Session clustering algorithm — groups related tab activity into logical work blocks
  • Storage in Supabase with per-user data isolation
  • Retention policy: raw activity data retained for 90 days, aggregated data indefinitely

2.3  Smart Daily Summary

End-of-Day AI Summary

  • AI analyses the day's tracked sessions alongside Calendar, Gmail, and Drive data
  • Generates a unified draft timesheet combining all sources
  • Highlights gaps: "You had 2 hours with no tracked activity between 2pm–4pm — what were you working on?"
  • Suggests entries with confidence levels (high/medium/low)
  • Feeds into the Phase 1 draft-and-approve workflow

2.4  Advanced Analytics & Reporting

Individual Reports

  • Time by project — weekly/monthly breakdown
  • Time by tool — which applications consume the most time
  • Productivity patterns — peak hours, focus time vs. fragmented time
  • Timesheet completion rate — percentage of working hours logged

Manager / Admin Reports

  • Team utilisation dashboard — at-a-glance view of who's logging and who isn't
  • Project time allocation — actual hours vs. budgeted across the team
  • Tool usage across the organisation — which platforms drive the most work
  • Under-logging alerts — flag team members consistently below threshold
  • Export to CSV / Google Sheets for further analysis

2.5  Notifications & Reminders

  • End-of-day reminder: "You have 3 hours unlogged today — review your draft?"
  • Weekly summary email: timesheet overview + missing days
  • Manager alerts: team members who haven't submitted timesheets by deadline
  • Configurable notification preferences per user

3. Technical Architecture

Extension

PlatformChrome (Manifest V3) — compatible with Edge, Brave, Arc
PermissionsactiveTab, tabs, idle, storage, alarms (no broad host permissions)
Data syncBatched upload every 5 minutes (not real-time, reduces bandwidth)
Offline supportLocal storage queue, syncs when connection restores
SizeLightweight (<500KB), no impact on browser performance

Backend Additions

APINew REST endpoints for activity ingestion, session queries, mapping rules
ProcessingSession clustering runs server-side on batched data
StorageSupabase — new tables for raw_activity, sessions, mapping_rules
AIProject inference uses the same AI model as Phase 1 (swappable)

Privacy & Security

  • The extension tracks URLs and page titles only — no page content, no keystrokes, no screenshots
  • Users can exclude specific domains (personal banking, social media, health portals)
  • All data encrypted in transit (TLS) and at rest
  • Per-user data isolation — no cross-user data access
  • Admin cannot see individual browsing history — only aggregated project time
  • Compliant with Australian Privacy Act and NZ Privacy Act 2020
Privacy by design: Managers see "Sarah logged 3 hours on MediaWorks via Jira" — they never see which specific URLs or pages Sarah visited. Individual browsing data is visible only to the user themselves.

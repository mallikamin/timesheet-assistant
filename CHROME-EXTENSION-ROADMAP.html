Phase 2 Roadmap

Chrome Extension

Prepared for: Thrive PR + Communications Date: April 2026 Version: 1.0

1. Executive Summary

Phase 1 (approved) captures work from Gmail, Google Calendar, and Google Drive.

Phase 2 adds a lightweight Chrome extension that silently tracks which websites your team uses and for how long. Combined with a per-user AI profile that learns each person's patterns over time, the system becomes dramatically smarter — suggesting time entries that are more accurate, faster to approve, and require less manual input.

The result: A timesheet assistant that captures work from every digital source — email, calendar, documents, and browser — and learns each team member's habits to reduce time logging from 15 minutes/day to under 2 minutes.

2. What We're Building

2A. Chrome Extension

A small, silent Chrome extension that tracks the active tab's domain name and page title while the user works. It does not read page content, capture screenshots, or log keystrokes.

2B. Smart Deduplication Engine

When a user edits a Google Doc, that activity appears in both Google Drive and Chrome (docs.google.com). When they read emails, it shows in Gmail and Chrome. Without deduplication, the system would suggest double entries.

The deduplication engine prevents this by cross-referencing all data sources before suggesting entries:

┌──────────────┐ ┌──────────────┐ ┌──────────────┐ ┌──────────────┐ │ Gmail │ │ Calendar │ │ Google Drive │ │ Chrome │ │ (emails) │ │ (meetings) │ │ (files) │ │ (browsing) │ └──────┬───────┘ └──────┬───────┘ └──────┬───────┘ └──────┬───────┘ │ │ │ │ └──────────┬───────┴──────────┬───────┘ │ │ │ │ ▼ ▼ │ ┌──────────────────────────────────────┐ │ │ DEDUPLICATION ENGINE │◄─────────────┘ │ │ │ 1. Time-window overlap detection │ │ 2. Domain ↔ Source matching │ │ 3. Activity fingerprinting │ │ 4. Source priority ranking │ └──────────────┬────────────────────────┘ │ ▼ ┌─────────────────┐ │ Claude AI │ │ │ │ Sees unified, │ │ deduplicated │ │ activity feed │ └────────┬────────┘ │ ▼ ┌─────────────────┐ │ Draft Entries │ │ (user approves) │ └────────┬────────┘ │ ▼ ┌─────────────────┐ │ Harvest │ └─────────────────┘

How Deduplication Works

ScenarioWhat Happens
User edits Google Doc in Chrome Drive wins. Chrome activity on docs.google.com is suppressed because Drive already has the specific file name and edit time. Chrome adds nothing new.
User reads Gmail in Chrome Gmail wins. Chrome activity on mail.google.com is suppressed. Gmail already has sender, subject, and timestamp.
User joins Google Meet from Calendar Calendar wins. Chrome on meet.google.com is suppressed. Calendar already has meeting name, duration, attendees.
User works on Canva for 2 hours Chrome wins. No other source captures Canva activity. This is the gap Chrome fills.
User browses LinkedIn + writes email about same client Both kept, merged. Claude sees "1h LinkedIn + email to Client X" and suggests one combined entry, not two.
Source Priority (highest to lowest): Calendar (most specific — named meetings with durations) → Gmail (subject lines map to clients) → Drive (file names map to projects) → Chrome (catch-all for everything else)

Automatic domain exclusions: When Gmail, Calendar, or Drive scanning is active, the extension automatically excludes mail.google.com, calendar.google.com, docs.google.com, drive.google.com, and meet.google.com from Chrome tracking. Zero duplicates from Google services.

2C. Per-User AI Profiles

Every person at Thrive works differently. A senior account manager's Chrome activity looks nothing like a junior designer's. The AI profile system learns each user's unique patterns and gets smarter over time.

What the AI Learns Per User

PatternExampleHow It Helps
Domain → Client mapping "When Sarah uses Canva, it's usually Fonterra (80%) or Afterpay (20%)" Suggests correct client automatically
Time-of-day patterns "Tariq does Afterpay work in the morning, AGL in the afternoon" Better guesses when evidence is ambiguous
Typical session durations "Designers average 2h on Canva per session, account managers 30min" More accurate time estimates
Approval patterns "James always edits 'Operations' entries to 'Business Development'" Learns from corrections, doesn't repeat mistakes
Tool-to-task mapping "Hootsuite = Social Media Management, WordPress = Content Creation" Maps browser tools to Harvest tasks
Day-of-week rhythms "Monday = team meetings + admin, Friday = reporting + timesheets" Context-aware suggestions
How it gets smarter: Every time a user approves, edits, or rejects a suggested entry, the system records the correction. After 2-3 weeks of use, each user's AI profile is calibrated to their specific work patterns. Accuracy improves from ~60% (day one) to ~90%+ (after training).

Privacy Guarantee

AI profiles are:

  • Per-user only — your profile is never shared with other users or managers
  • Metadata-based — learns from domains, time patterns, and approval history (never content)
  • Deletable — any user can reset their profile at any time
  • Transparent — users can see what their profile has learned ("Show my patterns")
  • Australian-compliant — same metadata-only architecture as Phase 1

6. Enterprise Deployment (50+ Users)

For Thrive's 50-60 person team, the Chrome extension is deployed via Google Admin Console — not the public Chrome Web Store. This means:

  • Centralized install — IT admin pushes the extension to all users at once
  • No user action needed — extension appears automatically in Chrome
  • Policy-managed — can be mandatory or optional per user group
  • Updates pushed silently — new versions deploy without user intervention
  • Removable by admin — can be rolled back instantly if needed

Thrive already uses Google Workspace, so this infrastructure is already in place.

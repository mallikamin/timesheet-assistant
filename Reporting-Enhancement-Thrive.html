Your Current Reporting Workflow

Based on the reporting templates and screenshots shared by your team, we understand that Thrive's current time and utilisation reporting involves several manual steps:

  • Time entry exports — detailed time data is exported from Harvest to Excel (date, client, project, task, notes, hours) for analysis and record-keeping
  • Forecast scheduling data — scheduled hours and resource allocations are viewed in the Harvest Forecast interface and referenced separately when building reports
  • Utilisation reports — actual hours (from Harvest) are manually cross-referenced against scheduled hours (from Forecast) to calculate utilisation rates, billable percentages, and capacity gaps per employee
  • Team roster — employee information including roles, locations, and departments is maintained and referenced alongside the above

The result is a reporting process that requires someone to pull data from two separate systems, combine them in Excel, and produce the final utilisation summary by hand. This is repeated weekly or monthly.

What the APIs Make Possible

The Harvest and Forecast APIs provide programmatic access to all the underlying data. When combined, they can replace the manual export-and-assemble workflow entirely.

Report Current Method With Automation
Detailed time entries Manual CSV export from Harvest On-demand or scheduled via Harvest API
Scheduled hours & resource allocation Viewed in Forecast UI, cross-referenced manually Pulled via Forecast API and merged automatically
Team utilisation (actual vs scheduled) Built manually in Excel from two sources Auto-generated — Harvest API returns both when linked
Billable percentage by employee Calculated manually per period Computed in real time from Harvest reports endpoint
Capacity & availability gaps Separate Forecast lookup + manual comparison Integrated view with flagging for under-capacity

Additional automated outputs can include:

  • Weekly or monthly reports delivered as Google Docs or PDF, auto-shared to managers
  • Hours breakdown by client and project — matching Harvest's project structure
  • Attention flags: users below target hours, pending approvals, missing time gaps
  • Forecast vs actual variance by project — are projects on track against scheduled allocations?
